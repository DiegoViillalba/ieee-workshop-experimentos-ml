Resumen

Presentamos un experimento reproducible para la clasificación binaria de tumores mamarios utilizando el dataset Breast Cancer Wisconsin. Empleando regresión logística con ponderación de clases, logramos recall = 0.976 y AUC-ROC = 0.994 sobre el conjunto de prueba, sosteniendo la hipótesis H₁ (recall ≥ 0.90). El experimento utiliza un pipeline reproducible basado en DVC y Weights & Biases, y documenta el proceso completo con Quarto para múltiples audiencias.

Palabras clave

machine learning, breast cancer, logistic regression, reproducibility, classification

1 Introducción

El diagnóstico temprano de cáncer de mama es uno de los factores más determinantes en la supervivencia de las pacientes. Los modelos de aprendizaje automático ofrecen una vía complementaria para asistir la interpretación de imágenes y datos clínicos (Street, Wolberg, y Mangasarian 1993).

Este trabajo presenta un experimento reproducible para clasificar tumores mamarios como malignos o benignos a partir de características morfológicas extraídas computacionalmente, con énfasis en el proceso experimental más que en la optimización de métricas.

1.1 Motivación

El dataset Breast Cancer Wisconsin (Street, Wolberg, y Mangasarian 1993; Mangasarian, Street, y Wolberg 1995) contiene mediciones de características del núcleo celular obtenidas a partir de imágenes digitalizadas de biopsias con aguja fina (FNA). Es un benchmark establecido que permite ilustrar los principios de diseño experimental en ML sin requerir infraestructura de GPU.

1.2 Contribuciones

Este trabajo:

  1. Formula una pregunta científica falsable con criterio cuantitativo.
  2. Implementa un pipeline reproducible usando DVC y Weights & Biases.
  3. Compara dos familias de modelos con análisis de estabilidad.
  4. Documenta todo el proceso con Quarto para múltiples audiencias.

2 Metodología

2.1 Dataset

Utilizamos sklearn.datasets.load_breast_cancer, que contiene 569 muestras con 30 características morfológicas numéricas y una etiqueta binaria: 0 = maligno (212 casos, 37.3%) y 1 = benigno (357 casos, 62.7%).

Código 
from sklearn.datasets import load_breast_cancer
import pandas as pd

bc = load_breast_cancer()
summary = pd.DataFrame({
    "Split":       ["Train", "Val", "Test", "Total"],
    "N muestras":  [384, 68, 114, 569],
    "% maligno":   ["37.2%", "37.1%", "37.7%", "37.3%"],
})
summary

La división es estratificada (stratify=y) para garantizar proporciones de clase similares en los tres splits (Pineau et al. 2021).

2.2 Preprocesamiento

Todas las características se escalan con StandardScaler, ajustado exclusivamente sobre el training set para evitar data leakage (Hastie, Tibshirani, y Friedman 2009).

2.3 Modelos

Se comparan dos familias:

Logistic Regression — modelo interpretable con regularización L2 y ponderación de clases (class_weight="balanced").

MLPClassifier — red neuronal de una capa oculta, tratada como modelo de mayor capacidad para comparación.

# Configuración baseline (configs/baseline.yaml)
model:
  type:         "logistic_regression"
  class_weight: "balanced"
  C:            1.0
  max_iter:     200

2.4 Métricas de evaluación

La métrica primaria es recall de la clase maligna, dado el costo asimétrico de los errores en diagnóstico clínico: un falso negativo (cáncer no detectado) tiene consecuencias graves, mientras que un falso positivo genera un estudio adicional.

\[\text{recall}_{\text{maligno}} = \frac{TP}{TP + FN}\]

Como métrica secundaria se usa AUC-ROC, que mide la capacidad discriminativa independientemente del umbral de decisión.

2.5 Hipótesis

H₀: \(\text{recall}_{\text{maligno}} < 0.90\) o \(\text{AUC-ROC} \leq 0.50\)

H₁: \(\text{recall}_{\text{maligno}} \geq 0.90\) y \(\text{AUC-ROC} > 0.50\)

2.6 Validación

Para garantizar robustez del resultado se realizan:

  1. Análisis de estabilidad con 3 seeds (7, 42, 99).
  2. Barrido de hiperparámetros con 9 configuraciones por modelo.
  3. Threshold tuning en el val set (umbral óptimo: 0.4).
  4. Diagnóstico train/test para detectar overfitting.

3 Resultados

3.1 Resultado principal

El modelo logró recall_malignant = 0.976 y AUC-ROC = 0.994, superando en ambas métricas al clasificador de mayoría. La hipótesis H₁ se sostiene.

3.2 Evaluación de H₁

De 42 tumores malignos en el test set, el modelo detectó correctamente 41 (TP = 41, FN = 1). Los 5 falsos positivos corresponden a casos benignos clasificados como malignos, lo que generaría estudios adicionales pero no representa un riesgo clínico inmediato.

3.3 Estabilidad del resultado

El recall es idéntico (0.9762) en las tres semillas, lo que indica que el resultado es determinista dado el split y los hiperparámetros. La varianza observada en AUC (σ = 0.0013) está dentro del rango aceptable.

3.4 Comparación Logistic vs MLP

Ambos modelos obtienen recall medio 0.968 con std 0.012 sobre 9 configuraciones cada uno. La diferencia en rendimiento es estadísticamente no significativa. La regresión logística se prefiere por su interpretabilidad directa.

4 Discusión

4.1 Interpretación del modelo

La regresión logística permite inspeccionar los coeficientes directamente. Los tres features más influyentes son radius error, worst texture y worst concave points, todos con coeficiente negativo: valores más altos indican mayor probabilidad de malignidad. Este patrón es consistente con el conocimiento médico sobre morfología tumoral.

4.2 Limitaciones

  1. Tamaño del dataset: 569 muestras es pequeño para generalizar a nuevas poblaciones. Los resultados son válidos como prueba de concepto.

  2. Distribución de clases: el desbalance (37/63) justifica el uso de class_weight="balanced", pero en producción el umbral óptimo dependería de las tasas de prevalencia reales de la población objetivo.

  3. Validación externa: el modelo no ha sido evaluado en datos de otra institución o cohorte demográfica diferente.

  4. Explicabilidad regulatoria: en muchos contextos clínicos se requieren métodos de explicabilidad certificados (GDPR, FDA) que van más allá del análisis de coeficientes.

4.3 Comparación de modelos

El MLP no supera de forma estadísticamente relevante a la regresión logística en este dataset. Esto es consistente con la literatura (Hastie, Tibshirani, y Friedman 2009): para datos tabulares de baja dimensión, los modelos lineales suelen ser competitivos con métodos más complejos.

Elegir MLP añadiría opacidad sin ganancia medible en recall, lo que no está justificado para un sistema de soporte a decisión clínica.

4.4 Implicaciones prácticas

Un modelo con recall = 0.976 significaría, en una cohorte de 1000 mujeres con 370 casos malignos, no detectar aproximadamente 9 casos. Según el contexto clínico, esto puede ser aceptable como herramienta de triaje inicial, pero no como diagnóstico definitivo.

4.5 Trabajo futuro

  • Validación en dataset externo.
  • Análisis de equidad por subgrupos demográficos.
  • Calibración de probabilidades para uso clínico.
  • Integración con sistemas de gestión de imágenes (PACS/RIS).

5 Conclusión

Este trabajo demuestra que un modelo de regresión logística con ponderación de clases puede alcanzar recall clínicamente relevante (≥ 0.90) en el dataset Breast Cancer Wisconsin, con resultados estables bajo variación de semillas. El pipeline reproducible implementado permite replicar exactamente los resultados y extender el experimento de forma controlada.

La comparación con MLP muestra que mayor complejidad no implica mayor rendimiento en este contexto, reforzando el valor de modelos interpretables como punto de partida en problemas de diagnóstico médico.

La estructura de documentación con Quarto permite comunicar el mismo experimento a audiencias diversas (papel académico, presentación, blog) desde un único source, reduciendo la fricción entre investigación y comunicación.
